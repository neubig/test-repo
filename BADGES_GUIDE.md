## Badge Types

The generator creates 5 different badge types:

### 1. Migration Status Badge
Shows overall migration state: "not started", "in progress", "nearly complete", or "complete"

**Example:**
- ![Migration Status](https://img.shields.io/badge/migration-in%20progress-yellow)

**Colors:**
- Red: Not started
- Yellow: In progress
- Light green: Nearly complete (90%+)
- Bright green: Complete

### 2. Progress Percentage Badge
Shows exact completion percentage based on files migrated

**Example:**
- ![Progress](https://img.shields.io/badge/migration%20progress-73%25-yellowgreen)

**Colors scale from red (0%) â†’ orange â†’ yellow â†’ light green â†’ green â†’ bright green (100%)**

### 3. Files Migrated Badge
Shows count of migrated files vs total files

**Example:**
- ![Files](https://img.shields.io/badge/files%20migrated-45%2F67-yellowgreen)

**Color matches completion percentage**

### 4. Issues Remaining Badge
Shows number of Python 3 compatibility issues still to fix

**Example:**
- ![Issues](https://img.shields.io/badge/issues%20remaining-23-orange)

**Colors:**
- Bright green: 0 issues (none)

- Red: 50+ issues

### 5. Python Version Badge
Simple indicator showing if code is Python 3 ready

**Example:**

- ![Python](https://img.shields.io/badge/python-2.x%20only-red)

## Tracking Progress Over Time

### Regular Updates

Regenerate badges after migration milestones:

```bash
# After fixing a batch of files
./py2to3 fix src/module1/
./py2to3 badges

# Commit updated badges
git add badges/
git commit -m "Update migration progress badges"
```

### Automated Updates

Add to your CI/CD pipeline:

```yaml
# .github/workflows/migration.yml
name: Update Migration Badges

on:
  push:
    branches: [main, migration/*]

jobs:
  badges:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.x'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Generate badges
        run: ./py2to3 badges
      - name: Commit badges
        run: |
          git config --local user.name "GitHub Action"
          git config --local user.email "action@github.com"
          git add badges/
          git diff --quiet && git diff --staged --quiet || git commit -m "chore: update migration badges [skip ci]"
          git push
```

## Technical Details

### Badge Format

Badges are generated as SVG (Scalable Vector Graphics):
- Vector format - scales perfectly at any size
- Small file size (typically <2KB each)
- Native browser support - no dependencies
- Compatible with all markdown renderers

### Data Source

Badges use migration statistics from:
1. `.migration_stats.json` file (if available from stats tracker)
2. Live verification scan (runs verifier if no stats found)

### Color Scheme

Colors match popular badge services like shields.io:
- Red: `#e05d44` - Problems, not started
- Orange: `#fe7d37` - Many issues
- Yellow: `#dfb317` - Some progress
- Yellow-green: `#a4a61d` - Good progress
- Green: `#97ca00` - Almost done
- Bright green: `#4c1` - Complete!
- Blue: `#007ec6` - Info
- Grey: `#555` / `#9f9f9f` - Neutral

## Troubleshooting

### Badges Show 0/0 Files

**Problem:** Badges show "0/0" or "no files"

**Solutions:**
1. Make sure you're in the project root directory
2. Run verifier first: `./py2to3 check .`
3. Collect stats first: `./py2to3 stats collect --save`
4. Make sure there are .py files in your project

### Badges Not Showing in GitHub

**Problem:** Badges don't display in GitHub README

**Solutions:**
1. Make sure badges are committed: `git add badges/ && git commit`
2. Push to GitHub: `git push`
3. Check file paths in markdown match actual file locations
4. Use relative paths: `badges/migration-progress.svg`

### Badge Colors Don't Update

**Problem:** Regenerating badges but colors stay the same

**Solutions:**
1. Clear browser cache (Ctrl+F5)
2. Check if badge files were actually updated: `ls -la badges/`
3. Verify stats are current: `./py2to3 stats show`
4. Run verifier to refresh data: `./py2to3 check .`

### Permission Errors

**Problem:** Can't write to badges directory

**Solutions:**
1. Check directory permissions: `ls -la badges/`
2. Create directory manually: `mkdir -p badges`
3. Use custom output directory: `./py2to3 badges --output ~/my-badges`

## FAQ

**Q: How often should I regenerate badges?**  
A: After each significant migration milestone (e.g., after fixing a module, weekly sprints, before team meetings).

**Q: Can I customize badge colors?**  
A: Currently colors are automatic based on progress levels. Edit `src/badge_generator.py` for customization.

**Q: Do badges work with GitHub, GitLab, Bitbucket?**  
A: Yes! SVG badges work in any markdown renderer that supports images.

**Q: Can I use these with private repositories?**  
A: Yes, badges are local files committed to your repo, so they work with private repos.

**Q: Do badges slow down README loading?**  
A: No, SVG files are tiny (<2KB each) and load instantly.

**Q: Can I use external badge services instead?**  
A: These are self-contained badges. For external hosting, consider shields.io endpoint badges.

**Q: How do badges differ from the dashboard?**  
A: Badges are static images for quick status. Dashboard is interactive HTML for detailed analysis.

**Q: Can I generate badges for multiple projects?**  
A: Yes, run the command in each project directory, or specify paths if implementing multi-project support.
